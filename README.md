# A_State

## Features

- Interactive Lua console
- Execute raw Lua code at runtime
- Load and execute `.lua` files from disk
- Dump Lua globals to file
- Trace Lua function calls with `hook call`
- Get immediate feedback from console execution, including prints and errors
- And more to come

## Usage

1. Build the project.
2. Place `A_State.dll` and `A_State_Injector.exe` in the same folder.
3. Run `A_State_Injector.exe`.
4. Choose an injection mode:
   - `1` - Auto-Inject: inject as soon as Alicia is found
   - `2` - Delayed Inject: wait a chosen number of seconds after Alicia is found
   - `3` - Manual Inject: wait for manual confirmation before injecting
5. Start the game after selecting the desired injection mode.
6. After injection, use the A_State console window to issue commands.

## Currently Available A_State Commands

### General

- `help` - show the command list
- `quit` - eject A_State cleanly
- `dump` - dump Lua globals to `A_State_Globals.txt`
- `load <file.lua>` - execute a Lua file from disk
- `print <expr>` - evaluate and print a Lua expression

### Hooks

- `hook call` - trace Lua function calls
- `hook native` - report native Lua load/runtime errors
- `hook stop` - stop tracing hooks
- `hook cooldown <ms>` - set aggregation window for repeated call entries

### Debug UI

- `open test`
- `open server`
- `open ranch`
- `open myhorse`
- `open injury`
- `open injurystatus`
- `open ai`
- `open skill`
- `open var`
- `open ghost`
- `open room`
- `close <name>`
- `close all`

### Utility

- `pos`
- `chat <msg>`
- `switchhook <0|1>` - `0` = safe hook, `1` = direct pointer
- any other input is executed as raw Lua code

## Requirements

- Windows
- Visual Studio 2017, 2019, or 2022 (full IDE or **Build Tools** edition) with the **Desktop development with C++** workload
- CMake 3.10+

## Building

Clone the repository and initialize the MinHook submodule:

```bash
git clone https://github.com/Chrommd/A_State
cd A_State
git submodule update --init --recursive
```

## Output Files

- `A_State_Diagnostics.txt` - startup diagnostics, address snapshot, and hook install status
- `A_State_Globals.txt` - generated by the `dump` command

## Notes

- If you intend to inspect the inner workings of native scripts through `hook call`, you should use injection mode `1` so A_State can intercept most scripts from the start.
- Scripts that are already loaded before injection cannot be intercepted retroactively, so their full path and filename may remain hidden in traces and errors.
- Current support targets the `tag10` version of the Alicia binary. Support for `tag7` may be added in the future.
- Windows Defender may flag A_State due to its DLL injection behavior. If injection fails or files get quarantined, add the A_State folder to your Windows Defender folder exclusions.
